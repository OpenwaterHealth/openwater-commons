# Request for Comments (RFC) Process

How to propose major changes to Openwater.

---

## üéØ When to Write an RFC

### Requires RFC:
- Major architectural changes
- Breaking API changes
- New platform features
- License/governance changes
- Significant process changes

### Doesn't Require RFC:
- Bug fixes
- Documentation improvements
- Minor feature additions
- Performance optimizations

**When in doubt, ask in [discussions](https://github.com/OpenwaterHealth/openwater-commons/discussions).**

---

## üìù RFC Template

See [rfcs/template.md](../../rfcs/template.md) for the full template.

**Key sections:**
- **Summary:** One-paragraph overview
- **Motivation:** Why is this needed?
- **Design:** How will it work?
- **Alternatives:** What else was considered?
- **Impact:** Who is affected?

---

## üîÑ RFC Lifecycle

### 1. Draft (Week 1)
- Author writes RFC
- File PR to `rfcs/` folder
- Name: `RFC-NNN-descriptive-title.md`
- Status: `Draft`

### 2. Comment Period (Week 2-3)
- Community provides feedback
- Author addresses questions
- Iterate on design
- Status: `In Discussion`

### 3. TSC Review (Week 4)
- TSC discusses in meeting
- Vote on acceptance
- Decision recorded in meeting notes
- Status: `Accepted`, `Rejected`, or `More Discussion Needed`

### 4. Implementation
- If accepted, create implementation issues
- Track progress
- Status: `Implementing`

### 5. Completion
- Implementation merged
- RFC marked complete
- Status: `Implemented`

---

## üìä RFC Status

- **Draft:** Being written
- **In Discussion:** Open for comments
- **TSC Review:** Awaiting decision
- **Accepted:** Approved, ready to implement
- **Rejected:** Not moving forward
- **Implementing:** Being built
- **Implemented:** Complete
- **Superseded:** Replaced by newer RFC

---

## üí° Tips for Great RFCs

1. **Start with "why":** Motivation is crucial
2. **Be specific:** Vague proposals get vague feedback
3. **Show alternatives:** What else did you consider?
4. **Think about impact:** Who is affected?
5. **Include examples:** Code samples, diagrams
6. **Get feedback early:** Draft RFC in discussions first

## ‚ùì FAQ

**Q: How long does the RFC process take?**
A: Typically 4-6 weeks from draft to decision.

**Q: Can anyone write an RFC?**
A: Yes! All community members can propose RFCs.

**Q: What if my RFC is rejected?**
A: Rejection means "not now" - you can revise and resubmit.

**Q: Can RFCs be amended?**
A: Yes, via a new RFC that references the original.
